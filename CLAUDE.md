# Project: Phaser Top-Down Space Shooter for a 5 yo boy

** This is work in progress, we should make it better each iteration **

## Stack

- **Phaser 3** (arcade physics, procedural textures — currently no external assets)
- **TypeScript** (strict)
- **Vite** for bundling

## Commands

```
npm run build   # type-check + production build
npm run dev     # local dev server
```

## Structure

```
src/game/
  constants.ts           # all numeric tuning values
  TextureFactory.ts      # procedural texture generation (incl. all ship variants + particle textures)
  ShipDefinitions.ts     # ShipDef interface + SHIP_DEFS array of 5 ships + DEFAULT_SHIP_KEY
  ChallengeDefinition.ts # ChallengeType, EnemyConfig, ChallengeDefinition + ALL_CHALLENGES array
  HUD.ts                 # score / challenge label / hull / center text + learning tracker row + math question
  InputHandler.ts        # keyboard + gamepad abstraction + controller vibration
  Player.ts              # player sprite, movement, firing, health/damage/respawn
  EnemyManager.ts        # enemy group, AI, continuous fixed-difficulty spawning, enemy bullets
  SoundManager.ts        # re-export barrel — keeps existing imports working; impl lives in audio/
  EffectsManager.ts      # all visual effects: explosions, trails, rings, sparks, muzzle flash
  LearningConfig.ts      # LearningSet interface + predefined sets (Hebrew, English, emojis…)
  LearningManager.ts     # drop logic, pickup physics group, collect/spell modes, win callback
  MathManager.ts         # math challenge: question generation, enemy answer labels, win callback
  MenuScene.ts           # challenge + ship selection screen (shown first)
  GameScene.ts           # slim orchestrator — wires everything together
  audio/
    SoundManager.ts      # public facade — wires AudioBus → SfxPlayer + MusicPlayer
    AudioBus.ts          # Web Audio graph: master / SFX / music gain chain + baked noise buffers
    MusicPlayer.ts       # clock-lookahead scheduler + space-echo delay bus
    MusicTrack.ts        # BPM, note patterns (edit here to change the music)
    SfxPlayer.ts         # one-shot SFX synthesis; add play*() here for new effects
```

## Conventions

- Add new tunable numbers to `constants.ts`
- New game systems get their own file in `src/game/`
- `GameScene` only does collision wiring and high-level sequencing
- No external asset files — textures are drawn with Phaser Graphics in `TextureFactory.ts`
- **No wave system** — enemies spawn continuously at fixed difficulty set by `EnemyConfig` in `ChallengeDefinition.ts`
- Sound is entirely procedural (Web Audio API) via `SoundManager` (re-export barrel → `audio/`); no audio asset files
- Player health is tracked in `Player` and surfaced via the `onHealthChanged` callback
- Gamepad vibration is triggered from `InputHandler` (`vibrateHit()` / `vibrateFire()`)
- Ship textures are 64×64 and generated by `drawCuteShip()` in `TextureFactory.ts` for every entry in `SHIP_DEFS`
- No full-screen camera flashes — effects use localised particles, rings, blooms, and camera shake only

## Effects System

`EffectsManager.ts` is the single place for all visual effects. `GameScene` holds one instance and calls it from collision callbacks and player event hooks.

**Particle textures** are registered in `TextureFactory.createTextures()` alongside all other textures:
`particle_spark`, `particle_smoke`, `particle_glow`, `particle_ring`, `particle_debris`.

**Other procedural textures (non-ship):**

- `enemy` — 44×44 green UFO saucer with cyan cockpit dome and engine port lights
- `asteroid` — 80×80 irregular polygon; blue-gray flat-shaded with light/shadow faces and three craters; spawns at random scales (0.6–1.1×)
- `nebula` — 512×512 soft purple/blue cloud blobs tiled as the deepest StarField layer (ADD blend)
- `bullet` / `enemyBullet` — 8×8 glowing orbs (cyan / orange) with soft outer halo

**Available effects (public API):**

| Method                                      | Trigger                                                  |
| ------------------------------------------- | -------------------------------------------------------- |
| `explodeEnemy(x, y)`                        | Bullet or rocket kills an enemy                          |
| `explodeAsteroid(x, y)`                     | Bullet or rocket destroys an asteroid                    |
| `playerHit(x, y)`                           | Player takes damage (via `Player.onHit`)                 |
| `playerDeath(x, y)`                         | Player health reaches 0 (via `Player.onDied`)            |
| `playerRespawn(x, y)`                       | Player finishes respawn delay (via `Player.onRespawned`) |
| `muzzleFlash(x, y)`                         | Player fires a bullet (via `Player.onFired`)             |
| `updateEngineTrail(x, y, rotation, active)` | Called every frame in `GameScene.update()`               |
| `updateRocketExhausts(rockets)`             | Called every frame in `GameScene.update()`               |
| `pickupCollect(x, y)`                       | Player collects a learning badge                         |
| `waveClear(x, y)`                           | All enemies in the wave are destroyed                    |

**Player callbacks used by EffectsManager:**

- `Player.onFired(x, y)` — gun-tip world position, emitted after every bullet
- `Player.onHit(x, y)` — player world position, emitted when damage is taken
- `Player.rotation` — sprite rotation getter (radians), used to aim the engine trail

**Design rule:** avoid full-screen `camera.flash()` calls — they are too harsh. Use localised blooms (`spawnFlash`), shockwave rings (`spawnRing`), particle bursts, and `camera.shake()` instead.

## Ship System

`ShipDefinitions.ts` is the single source of truth for player ship variants:

- Each `ShipDef` holds a texture `key`, display `label`, and five colours (`bodyColor`, `bodyDark`, `wingColor`, `cockpitColor`, `engineColor`)
- `SHIP_DEFS` contains 5 ships; `DEFAULT_SHIP_KEY` points to the first one
- All ship textures are registered at startup in `TextureFactory.createTextures()` via a loop over `SHIP_DEFS` — no other files need changing when adding a ship
- `Player` accepts an optional `shipKey` constructor parameter (5th arg, defaults to `DEFAULT_SHIP_KEY`)
- Ship sprites are **64×64**; hitbox is a 44-diameter circle offset (10, 10); gun-tip muzzle 30 px ahead of centre
- **To add a new ship:** append a `ShipDef` object to `SHIP_DEFS` in `ShipDefinitions.ts` — texture generation and the player constructor both pick it up automatically
- **To wire up ship selection:** pass the chosen `ShipDef.key` as the 5th argument to `new Player(...)` in `GameScene`

## Sound System

`src/game/audio/` contains the modular audio subsystem. The top-level `SoundManager.ts` is a one-line re-export barrel so existing callers (`GameScene`, `Player`, etc.) need no import changes.

| File                    | Responsibility                                                                           |
| ----------------------- | ---------------------------------------------------------------------------------------- |
| `audio/SoundManager.ts` | Public facade — null-safe delegation to SfxPlayer / MusicPlayer                          |
| `audio/AudioBus.ts`     | Web Audio graph: master → SFX / music gain chain; pre-baked noise buffers for drums      |
| `audio/MusicPlayer.ts`  | Clock-lookahead scheduler + space-echo delay bus; loops `MusicTrack` patterns            |
| `audio/MusicTrack.ts`   | BPM, `LOOP_BEATS`, and all note pattern arrays — **only file to edit for music changes** |
| `audio/SfxPlayer.ts`    | One-shot SFX via Web Audio oscillators / noise; add `play*()` methods here for new SFX   |

**To add a new SFX:** add a `play*()` method in `SfxPlayer.ts`, then delegate it through `audio/SoundManager.ts` and expose it on the barrel `SoundManager.ts`.

**To change the music:** edit note pattern arrays or `BPM` in `MusicTrack.ts` — no other file needs touching.

**To add a second music layer:** instantiate a second `MusicPlayer` with a different track in `audio/SoundManager.ts`.

## Visual Style

The game targets a **cartoon/chunky arcade** look — bold outlines, saturated colours, dramatic effects.

**Background** — `StarField.ts` renders 4 parallax tile-sprite layers (depth −10 → −7):

| Layer key    | Depth | Parallax | Description                                                |
| ------------ | ----- | -------- | ---------------------------------------------------------- |
| `nebula`     | −10   | 0.018    | 512×512 purple/blue ADD-blend cloud blobs; breathes slowly |
| `stars_far`  | −9    | 0.05     | 512×512, 120 dim dots — various tones                      |
| `stars_mid`  | −8    | 0.14     | 400×400, 40 medium-brightness dots                         |
| `stars_near` | −7    | 0.30     | 300×300, 15 bright stars with glow halos; twinkles         |

**HUD font** — `"Impact", "Arial Black", sans-serif` (arcade-bold). Hebrew text uses `"Arial Hebrew", "Arial Black"`.
HUD panel colour: `#080820` with a `#00e5ff` cyan border glow.

**Asteroid spawning** — 9 asteroids at game start, each at a random scale 0.6–1.1× with hitbox adjusted to match.

## Learning System

Enemies have a configurable chance (`LEARNING_DROP_CHANCE` in `constants.ts`) to drop a collectible badge when killed. The player flies into a badge to collect the item it displays. All items in the active set are shown at the top of the screen — **gold** when collected, **dim** when still missing. Collecting everything triggers a win screen.

**To change what kids learn:** edit `LearningConfig.ts` — the only file you need to touch:

- Add a new `LearningSet` object (any `items: string[]` array works — letters, words, emojis, numbers)
- Point `ACTIVE_LEARNING_SET` at it

**Drop mechanics:** `LearningManager.tryDrop(x, y)` is called by `GameScene` after every enemy kill (bullets and rockets). Only items not yet collected can drop. Pickups auto-expire after 20 s.

**HUD tracker:** `HUD.initLearningTracker(items, rtl?)` builds the top row once on scene create. `HUD.updateLearningTracker(collected)` refreshes it whenever an item is picked up. RTL flag reverses display order (used for Hebrew).
